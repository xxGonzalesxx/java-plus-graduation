# Explore With Me (EWM)
Это приложение, позволяет пользователям делиться информацией о событиях и находить компанию для участия в них.

## Этапы разработки

### Этап 1: Сервис статистики (Stats Service)

Реализована инфраструктура для сбора и анализа данных о посещаемости ресурсов.
- **Сбор данных:** Создано хранилище для регистрации обращений к эндпоинтам.
- **Аналитика:** Реализован расчет уникальных просмотров (по IP) и агрегация статистики за произвольный период.
- **Технологии:** Spring Boot, PostgreSQL, Docker.
- **Взаимодействие:** Разработан HTTP-клиент для интеграции статистики в основное приложение.

### Этап 2: Основной сервис (Main Service)

Реализована вся бизнес-логика приложения согласно спецификации API.
- **Публичный API:** Поиск событий с динамической фильтрацией (использован QueryDSL).
- **Приватный API:** Управление собственными событиями, участие в чужих мероприятиях, система подтверждения заявок.
- **Административный API:** Модерация контента, управление справочниками категорий и пользователями.
- **Интеграция:** Основной сервис в реальном времени обращается к Сервису статистики для получения данных о просмотрах.
- **Валидация:** Реализована сложная многоуровневая проверка бизнес-правил (проверка временных интервалов, лимитов участия, статусов модерации).

### Этап 3: Дополнительный функционал (Feature: Comments)

Проект расширен системой комментариев для повышения вовлеченности пользователей.
- **Возможности:** Пользователи могут оставлять комментарии к опубликованным событиям.
- **Модерация:** Реализованы статусы комментариев (`PENDING`, `PUBLISHED`, `REJECTED`).
- **Безопасность:** Добавлены проверки владения (редактировать комментарий может только автор).
- **Оптимизация:** Использование FetchType.LAZY и исключение рекурсивных вызовов в сущностях для обеспечения высокой производительности БД.

## Спецификация для реализации Comments: 

### Административный API:

`GET/admin/comments` - Поиск комментариев с фильтрацией
_Список параметров для фильтрации комментариев:_
* text - текст для поиска по содержимому комментария;
* List<Long> User - поиск по списку авторов комментариев;
* Long Event - поиск по идентификатору события;
* Enum status - поиск по статусу комментария (опубликован или нет);
* LocalDateTime rangeStart - дата и время не раньше которых оставлены комментарии;
* LocalDateTime rangeEnd - дата и время не позже которых оставлены комментарии;
* Integer from - количество комментариев, которые нужно пропустить для формирования текущего набора;
* Integer size - количество комментариев в наборе.

`PATCH/admin/comments/{commentId}` - обновление статуса комментария (опубликован или отклонен) по идентификатору

`DELETE/admin/comments/{commentId}` - удалить любой комментарий по идентификатору

-------------------------------------------------------------------------

### Публичный API:

`GET/comments` - Поиск опубликованных комментариев с фильтрацией
_Список параметров для фильтрации комментариев:_
* text - текст для поиска по содержимому комментария;
* Long Event - поиск по идентификатору события;
* LocalDateTime rangeStart - дата и время не раньше которых оставлены комментарии;
* LocalDateTime rangeEnd - дата и время не позже которых оставлены комментарии;
* Integer from - количество комментариев, которые нужно пропустить для формирования текущего набора;
* Integer size - количество комментариев в наборе;
* String sort - сортировка по дате.

`GET/comments/{commentId}` - поиск опубликованного комментария по идентификатору

-------------------------------------------------------------------------

### Приватный API:

`POST/users/{userId}/events/{eventId}/comment` - добавление комментария пользователем к опубликованному событию

`GET/users/{userId}/comments` - получение всех комментариев пользователя при условии, что он автор

`GET/users/{userId}/events/{eventId}/comments` - получение всех комментариев пользователя по идентификатору события при условии, что он автор

`GET/users/{userId}/comments/{commentId}` - получение комментария пользователя при условии, что он автор

`PATCH/users/{userId}/comments/{commentId}` - обновление комментария пользователя при условии, что он автор

`DELETE/users/{userId}/comments/{commentId}` - удаление комментария пользователя при условии, что он автор
